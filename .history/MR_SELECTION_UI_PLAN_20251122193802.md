# План реализации UI для выбора MR

## Текущий статус

✅ **Backend готов:**
- Классификация MR по типам (8 категорий)
- Баллы сложности (1-5)
- Теги стека
- API endpoints для фильтрации и рекомендаций
- Поддержка нескольких MR в одной сессии

⏳ **Frontend нужно обновить:**
- Добавить интерфейс выбора MR при создании сессии
- Показывать фильтры по типу, баллам, стеку
- Отображать рекомендации на основе грейда
- Показывать прогресс-бар выбранных баллов

### 2. Компоненты

#### MRSelector.jsx
- Выбор грейда (junior/middle/senior)
- Выбор стека (теги)
- Кнопка "Получить рекомендации"
- Прогресс-бар выбранных баллов
- Список MR с чекбоксами
- Фильтры

#### MRCard.jsx
- Отображение одного MR
- Тип, баллы, теги
- Чекбокс для выбора
- Предпросмотр diff (опционально)

### 3. API вызовы

```javascript
// Получить рекомендации
GET /api/mr/recommend?target_grade=middle&stack_tags=python,backend

// Получить список с фильтрами
GET /api/mr/list?mr_type=bugfix&min_complexity_points=2&stack_tag=python

// Создать сессию с несколькими MR
POST /api/reviewer/sessions
{
  "candidate_name": "...",
  "mr_ids": [1, 3, 5]
}
```
